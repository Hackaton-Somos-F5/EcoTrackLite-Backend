# Historia de Usuario US-003

## Crear modelo para los residuos
**Como** desarrollador backend
**Quiero** tener schemas Pydantic para la entrada y salida de datos
**Para** que FastAPI valide automáticamente los datos y el frontend reciba respuestas con formato consistente.

## Criterios de Aceptación

- ResiduoCreate valida que tipo sea uno de los valores del Enum
- ResiduoCreate valida que peso y volumen sean mayores que 0 con Field(gt=0)
- ResiduoResponse incluye id y fecha_registro además de los campos de creación
- FastAPI rechaza automáticamente datos inválidos con error 422
