# Historia de Usuario US-006

## Tests de endpoints de residuos
**Como** desarrollador backend
**Quiero** tener tests automatizados para POST y GET /residuos
**Para** garantizar que funcionan correctamente y alcanzar el 75% de cobertura.

## Criterios de Aceptación

- Los tests utilizan una base de datos SQLite en memoria o temporal para no ensuciar la de desarrollo
- Se incluye un test para el registro exitoso (`POST /colegios/{id}/residuos`)
- Se incluye un test para las validaciones fallidas (peso < 0)
- Se incluye un test para el filtrado de lista (`GET /colegios/{id}/residuos`)
- La cobertura de tests del módulo de residuos es de al menos el 75%

## Tareas

| Código | Nombre |
|--------|--------|
| TK-006-01 | Configurar `conftest.py` con el cliente de pruebas y base de datos de test |
| TK-006-02 | Crear `tests/test_residuos.py` con pruebas para POST y GET |
| TK-006-03 | Ejecutar tests con `pytest` y verificar que todos pasan |
| TK-006-04 | Generar informe de cobertura con `pytest-cov` |
