# Historia de Usuario US-008

## Registro de Nuevo Colegio
**Como** administrador del sistema
**Quiero** registrar un nuevo colegio en la plataforma con sus datos básicos
**Para** que el centro educativo pueda empezar a gestionar sus residuos.

## Criterios de Aceptación

- Existe el endpoint `POST /colegios`
- El cuerpo de la petición debe seguir el esquema `ColegioCreate` basado en `tables.md`:
    - `nombre`: VARCHAR(150), obligatorio
    - `direccion`: VARCHAR(200), obligatorio
    - `ciudad`: VARCHAR(100), obligatorio
    - `telefono`: VARCHAR(20), obligatorio
- El sistema debe asignar automáticamente un `id` y la `fecha_creacion`.
- Si el registro es exitoso, devuelve un código `201 Created` con el objeto completo.
- Se validan longitudes máximas y campos no vacíos.

## Tareas

| Código | Nombre |
|--------|--------|
| TK-008-01 | Implementar la lógica de creación en `services/colegio.py` |
| TK-008-02 | Crear el endpoint `POST /colegios` en `routes/colegio.py` |
| TK-008-03 | Registrar el router de colegios en `main.py` |
| TK-008-04 | Verificar el registro correcto mediante Swagger UI e inspección de DB |
