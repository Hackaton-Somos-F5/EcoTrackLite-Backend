# US-009: Asociación de Residuos con Categorías

Como usuario de un colegio, quiero que al registrar un residuo deba seleccionar una categoría existente para clasificar correctamente los desechos de la institución.

## Criterios de Aceptación

- [ ] Al registrar un residuo (`POST /residuos`), el campo `tipo` debe ser reemplazado o vinculado a un `categoria_id` válido.
- [ ] La respuesta al obtener residuos (`GET /residuos`) debe incluir la información de la categoría asociada (nombre y color).
- [ ] Si se intenta registrar un residuo con un `categoria_id` inexistente, devuelve un error 404.
- [ ] Integración con la tabla de alertas: los umbrales de alerta ahora deben configurarse por `categoria_id`.

## Tareas

- [ ] Modificar el modelo `Residuo` para añadir la relación (Foreign Key) con `Categoria`.
- [ ] Actualizar los schemas de `Residuo` para incluir el ID de la categoría.
- [ ] Modificar el servicio de registro de residuos para validar la existencia de la categoría.
- [ ] Realizar una migración de datos si es necesario (asociar tipos antiguos como "papel" a la nueva categoría "Azul").
