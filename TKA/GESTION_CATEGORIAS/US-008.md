# US-008: Gestión de Categorías de Residuos

Como administrador del sistema, quiero poder gestionar las categorías de residuos (nombre, color, descripción) para que el registro de residuos sea preciso y visualmente intuitivo según la normativa escolar.

## Criterios de Aceptación

- [ ] Existe un endpoint `POST /categorias` para crear una nueva categoría con: nombre (string), color (string/hex), y descripción (text).
- [ ] Existe un endpoint `GET /categorias` que devuelve el listado completo de categorías disponibles.
- [ ] Cada categoría tiene un `id` único autoincremental.
- [ ] No se permiten categorías con el mismo nombre (el campo debe ser único).
- [ ] El color debe almacenarse para ser usado en la interfaz del frontend (ej: "Azul", "Rojo", o código Hex).

## Tareas

- [ ] Definir el modelo `Categoria` en SQLAlchemy con los campos solicitados.
- [ ] Crear los schemas de Pydantic para la creación (`CategoriaCreate`) y respuesta (`CategoriaResponse`).
- [ ] Implementar el servicio para la lógica de negocio de categorías.
- [ ] Crear las rutas en `/routes/categoria.py` e incluirlas en `main.py`.
- [ ] Crear un script de inicialización o "seed" con las categorías escolares estándar (Azul, Amarillo, Verde, Marrón, Gris, Rojo).
