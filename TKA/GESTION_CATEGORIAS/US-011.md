# US-011: Filtrado y Consulta de Residuos por Categoría

Como personal administrativo de un colegio, quiero poder filtrar el listado de residuos por su categoría específica (Azul, Amarillo, Verde, etc.) para realizar auditorías rápidas sobre el volumen de materiales recolectados por tipo.

## Criterios de Aceptación

- [ ] El endpoint `GET /colegios/{id}/residuos` debe aceptar un parámetro de consulta (query parameter) opcional llamado `categoria_id`.
- [ ] Si se proporciona un `categoria_id`, la lista devuelta debe mostrar únicamente los residuos que pertenecen a esa categoría para ese colegio.
- [ ] Si no se proporciona el parámetro, el endpoint debe devolver el listado completo (comportamiento actual).
- [ ] La respuesta debe seguir incluyendo la información detallada de la categoría (nombre y color) para cada elemento filtrado.
- [ ] El filtrado debe ser compatible con otros filtros existentes (como el filtro de `estado`).

## Tareas

- [ ] Actualizar la función `get_residuos_by_colegio` en el servicio de residuos para aplicar el filtro dinámico.
- [ ] Modificar el endpoint en `routes/residuo.py` para exponer el parámetro `categoria_id`.
- [ ] Validar en la documentación de Swagger que el filtro aparece correctamente como opcional.
- [ ] Crear pruebas de integración en `tests/test_residuos.py` que verifiquen que el filtrado por categoría devuelve el número correcto de registros.
